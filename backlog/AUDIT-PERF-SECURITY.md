## Executive Summary
- **Severity counts (after dedup):** 4 high, 9 medium, 12 low вЂ” 25 actionable items.
- **Most urgent:** `SEC-01` вЂ” the FinDesk fiscal registry (`finRegistry.ts`) takes `tenantId` from the
  client on both reads and writes, allowing any authenticated user to read another tenant's tax rates /
  chart-of-accounts and any admin to write tax rates into a victim tenant. Live cross-tenant break in a
  paying-client prod app; fix first, effort S.
- **Theme behind "pages always reload":** (a) no client-side session/query cache and (b) the 642 KB eager
  main bundle вЂ” both in Batch 2.
- **Total estimated effort:** ~13 S + ~9 M + ~3 L в‰€ 4вЂ“6 focused working days.
- **Sequencing:** ship Batch 1 (security, smallest blast radius) before any perf work.

## BATCH 1 вЂ” Critical security & cross-tenant data-leak fixes

### SEC-01 В· finRegistry tenant-isolation break (read + write) вЂ” **HIGH В· S**
*(Consolidates: LEAK-finregistry-client-tenantid, LEAK-finregistry-taxrate-by-id, AUTHZ-finregistry-cross-tenant-read/write, SEC-finregistry-cross-tenant-read/write)*
- **Why:** Any authenticated user can read another tenant's fiscal config and any owner/admin can poison
  another tenant's VAT/tax rates вЂ” the core invariant (tenantId from session only) broken in one place.
- **Fix:**
  1. GET `/tax-rates` and `/chart-of-accounts`: delete `tenantIdParam`; always `resolvedTenantId = user.tenantId`.
     Keep `or(isNull(table.tenantId), eq(table.tenantId, user.tenantId))` so global+own rows still show.
  2. GET `/tax-rates/:id`: add `and(eq(finTaxRates.id, id), or(isNull(finTaxRates.tenantId), eq(finTaxRates.tenantId, user.tenantId)))`, 404 otherwise.
  3. POST `/tax-rates`: remove `tenantId` from `createTaxRateSchema`; always insert `tenantId: user.tenantId`.
- **Files:** `server/routes/finRegistry.ts` (~34, 56, 72вЂ“86, 89, 121, 155вЂ“157)
- **Acceptance:** e2e (tenant A token): `GET /tax-rates?tenantId=<B>` в†’ only A's+global rows; `GET /tax-rates/<B-id>` в†’ 404; `POST {tenantId:<B>}` в†’ row created under A (or 400). Fails on old code, passes on fix.

### SEC-02 В· No rate limiting on login / 2FA / password endpoints вЂ” **HIGH В· M**
- **Why:** Both login routes (+2FA-verify, reset/change-password) accept unlimited guesses; only limiter is
  an in-memory Map that resets per serverless instance вЂ” full-speed brute force / credential stuffing.
- **Fix:** DB-backed rate limiter keyed by (email+IP), `login_attempts` table. ~5 attempts/15 min, exp backoff,
  `429` + `Retry-After`. Apply to `/api/auth/login`, `/api/business/auth/login`, 2FA verify, reset/change-password.
- **Files:** `server/routes/auth.ts` (~106), `businessAuth.ts` (~47), `auth/twoFactor.ts` (~116),
  new `server/db/schema/loginAttempts.ts` + migration + `index.ts` export, new `server/lib/rateLimit.ts`.
- **Acceptance:** 6 failed logins for one email в†’ 6th returns `429` + `Retry-After`; persists across cold starts.
  Migration prefix > max on origin/main; `db:reset && db:seed` green.

### SEC-03 В· Global error handler leaks raw `err.message` to clients вЂ” **MEDIUM В· S**
- **Why:** Every uncaught exception serializes Postgres/SQL fragments, table/column names, fs paths into the
  500 body across all routers вЂ” free schema/stack recon.
- **Fix:** `app.onError` в†’ `{ error: "internal_error" }` (+ optional correlation id) in prod; `err.message`
  only when `NODE_ENV !== "production"`. Same for `/api/health` + `/api/health/db`.
- **Files:** `server/app.ts` (98вЂ“101, 223, 247)
- **Acceptance:** prod в†’ body `{"error":"internal_error"}` no SQL/path; dev still shows message. Test asserts prod body.

### SEC-04 В· PAR self-approval via role-routed step вЂ” **MEDIUM В· S**
- **Why:** A requestor who also holds the approver role (or is tenant admin/manager w/ implicit par_admin) can
  approve the role-routed step в†’ defeats segregation of duties (violates PAR-CORE.md:324 + T-PAR-107-3).
- **Fix:** In `approveParStep` (+ reject/request-changes/reapprove), after loading PAR, return
  `403 self_approval_forbidden` when `par.requestedByUserId === userId` вЂ” for role-routed steps too.
- **Files:** `server/lib/par/parApprovals.ts` (~140вЂ“200, 597), `server/routes/par/submit.ts`, `canActOnApproval.ts`
- **Acceptance:** T-PAR-107-3 extended: requestor-who-is-approver approving own role-routed step в†’ 403. e2e-par-100 green.

### SEC-05 В· FinDesk GDPR export/anonymize uses wrong RBAC + destructive scoping вЂ” **MEDIUM В· M**
- **Why:** Export/anonymize gate on CRM `users.role`, not `requireFinRole` в†’ a fin-`viewer` platform-admin can
  export student PII / mass-anonymize; `updatedAt < cutoff` can wipe PII of still-active students.
- **Fix:** Add `requireApp('business')` + `requireFinRole('owner')`; replace `user.role` checks. Base "inactivity"
  on a real activity signal, require explicit `confirm` flag, write to audit log.
- **Files:** `server/routes/finGdpr.ts` (27, 55, 129, 140вЂ“148)
- **Acceptance:** fin-`viewer` в†’ 403; fin-`owner` (CRM roleв‰ admin) в†’ 200; anonymize without `confirm:true` в†’ 400;
  active student not anonymized; audit row written. Tests cover all four.

## BATCH 2 вЂ” High-impact performance: "pages always reload / slow loading"

### PERF-01 В· Lazy-load all routes + vendor manualChunks (kill the 642 KB main bundle) вЂ” **HIGH В· M**
*(Consolidates: PERF-main-chunk-642kb, no-manualchunks, recharts-eager, jspdf-eager, giant-route-switch)*
- **Why:** Every first visit downloads the entire 3 MB app (642 KB gzip, 6.4Г— the В§3.4 budget) incl. all
  PAR/FinDesk pages + recharts + jspdf before render.
- **Fix:**
  1. Convert ~47 eager page imports in `src/App.tsx` to `lazy()` + shared `<Suspense fallback>` (13 already done).
  2. Add `build.rollupOptions.output.manualChunks` to `vite.config.ts`: `vendor-react`, `vendor-charts`(recharts), `vendor-pdf`(jspdf).
  3. Make jspdf dynamic in `parPdf.ts`/`finInvoicePdf.ts`/`certificateRender.ts`: `const { jsPDF } = await import("jspdf")` (mirror `poPdf.ts:103`).
- **Files:** `src/App.tsx` (7вЂ“84, 117вЂ“236), `vite.config.ts`, `src/lib/parPdf.ts:21`, `finInvoicePdf.ts:16`, `certificateRender.ts:11`
- **Acceptance:** entry chunk gzip в‰¤ ~150 KB; recharts+jspdf only in own/route chunks (grep entry в†’ 0 hits); per-route chunks emitted. `npm run smoke` green.

### PERF-02 В· Single shared session/roles cache (stop refetch storm) вЂ” **MEDIUM В· L**
*(Consolidates: CACHE-no-client-cache-refetch-storm, guard-shell-double-session-fetch, usesession-no-context-refetch-storm)*
- **Why:** Direct cause of "se Г®ncarcДѓ mereu paginile." `useSession`/`useBusinessSession`/`useParRoles` are
  per-component hooks, no cache; one FinDesk navigation fires `/api/business/auth/me` up to **3Г—** + `/api/par/me`,
  each from a cold full-screen spinner, on every hash change.
- **Fix:**

  2. Follow-up: `@tanstack/react-query`, `staleTime ~60s` for `/me` + `/par/me`.
- **Files:** `src/App.tsx`, new `src/contexts/SessionProvider.tsx`, `useSession.ts`, `useBusinessSession.ts`, `useParRoles.ts`, `BusinessGuardPage.tsx:23`, `BusinessShell.tsx:302/305`, `FinLayout.tsx:40`, `AppShell.tsx:196`
- **Acceptance:** navigating 3 business pages fires `/me` **once** (cached), no spinner re-flash. Guarded routes still fail closed.

### PERF-03 В· Immutable Cache-Control on hashed static assets вЂ” **LOW В· S**

- **Fix:** `headers` rule in `scripts/build-vercel.mjs` config.json (and/or `vercel.json`): `/assets/(.*)` в†’
  `Cache-Control: public, max-age=31536000, immutable`; `/index.html` + `/sw.js` в†’ `no-cache`.
- **Files:** `scripts/build-vercel.mjs` (~73вЂ“85), `vercel.json`
- **Acceptance:** `/assets/*.js` в†’ `вЂ¦immutable`; `/index.html` в†’ `no-cache`; new deploy still picked up.

## BATCH 3 вЂ” Remaining security hardening (medium/low)

### SEC-06 В· Rate/quota limit on AI extraction & file-upload endpoints вЂ” **MEDIUM В· M**
- **Why:** `/api/par/ai-prefill` + `/api/fin/captures(/batch)` invoke paid LLM gated only by `requireAuth` в†’
  unbounded spend + serverless concurrency exhaustion (max:1 pool).
- **Fix:** Per-tenant + per-user quota (reuse SEC-02 store) on AI + upload endpoints; cap batch size; `429` over limit.
- **Files:** `server/routes/parAiPrefill.ts:27`, `finCaptures.ts` (~750/796/892), `server/lib/rateLimit.ts`
- **Acceptance:** over per-tenant hourly AI cap в†’ `429`; over batch cap в†’ 400. Test invokes past limit, asserts 429.

### SEC-07 В· Upload size cap (memory-exhaustion DoS) вЂ” **LOW В· S**
- **Why:** Statement/capture uploads buffer the whole file into memory + run exceljs before any size guard в†’ OOM.
- **Fix:** Check `file.size` immediately; reject >10вЂ“15 MB with `413` before `arrayBuffer()`/`xlsx.load`. Per-file + total/count cap on multi-file path.
- **Files:** `server/routes/finStatement.ts` (142вЂ“158), `finCaptures.ts` (778вЂ“808)
- **Acceptance:** 20 MB file в†’ `413` before parsing; multi-file over count cap в†’ 400.

### SEC-08 В· Login account-enumeration via timing & response shape вЂ” **LOW В· M**
- **Why:** Existing emails run bcrypt (delay), unknown return early; businessAuth returns distinct codes (403 wrong_app / 401 disabled / 401 invalid) в†’ enumeration.
- **Fix:** Always run constant-time verify vs dummy hash; single generic `invalid_credentials` 401 at login step. Pair with SEC-02.
- **Files:** `server/routes/businessAuth.ts` (54вЂ“61, 72вЂ“78), `auth.ts` (111вЂ“122)
- **Acceptance:** latency unknownв‰€known; all failures return identical 401 at login step.

### SEC-09 В· Upload content-type sniffing (don't trust extension/MIME) вЂ” **LOW В· M**
- **Why:** Classification relies on attacker-controllable filename/MIME; client MIME stored verbatim in `data:` URL.
- **Fix:** Validate magic bytes (`%PDF`, PNG/JPEG, XLSX zip) + extension; derive stored mimeType from sniffed type.
- **Files:** `server/routes/finStatement.ts:148/182`, `finCaptures.ts:567/572/577`
- **Acceptance:** `.pdf`-named non-`%PDF` file rejected/reclassified; stored mimeType matches sniffed.

### SEC-10 В· Rotate session token after 2FA completion вЂ” **LOW В· M**
- **Why:** Pre-2FA token reused post-2FA (flag flipped on same row) в†’ leaked pending token becomes full session.
- **Fix:** On TOTP verify, revoke pending session + mint fresh `randomBytes` token, update cookie.
- **Files:** `server/routes/auth/twoFactor.ts` (162вЂ“166), `server/auth/session.ts` (26вЂ“42)
- **Acceptance:** cookie token differs before/after 2FA; old pending token rejected after verify.

### SEC-11 В· Validate PAR project approver userIds against tenant membership вЂ” **LOW В· S**
- **Why:** `setProjectApprovers` inserts arbitrary userIds w/o membership check (data-integrity gap; verified NOT a cross-tenant leak).
- **Fix:** Reuse parDoa pattern: reject any `userId` not in `parMembers` for tenant before delete/insert tx.
- **Files:** `server/lib/par/projectApprovers.ts` (68вЂ“83), `parProjects.ts` (40, 55)
- **Acceptance:** non-member UUID в†’ 400 no rows; valid members в†’ 200.

## BATCH 4 вЂ” Medium performance: DB indexes, N+1 queries, render perf

### PERF-05 В· Batch statement auto-match UPDATEs (N+1 writes) вЂ” **MEDIUM В· M**

- **Files:** `server/routes/finStatement.ts` (322вЂ“341)
- **Acceptance:** same final matchStatus rows, в‰¤2 UPDATE statements (assert via spy).

### PERF-07 В· Batch monthly depreciation reads/writes вЂ” **LOW В· M**
